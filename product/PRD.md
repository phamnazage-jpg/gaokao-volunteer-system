## 2. 产品概述

### 2.1 产品定位

**一句话描述**：为高考考生提供专业化、数据驱动、政策合规的志愿填报辅助的AI系统。

**核心定位**：

- 不是替代人工规划师，而是**AI赋能的规划助手**
- 不是简单的信息查询，而是**智能决策支持**
- 不是通用AI对话，而是**垂直领域的专业服务**

### 2.2 目标用户

#### 主要用户画像

**用户A：普通工薪家庭考生家长**

- 年龄：40-50岁
- 职业：工薪阶层/小企业主
- 痛点：信息不对称，不知道如何选，怕选错
- 需求：专业建议、风险提示、操作简单
- 付费意愿：中等（几百元以内可接受）

**用户B：县城/农村考生**

- 信息获取渠道有限
- 对政策不了解
- 需要标准化的、可靠的建议
- 付费意愿：低（倾向于免费或低价）

**用户C：一线城市知识型家长**

- 有信息获取能力
- 但缺少系统化的方法论
- 需要数据支持和验证工具
- 付费意愿：高（为专业服务付费）

#### 用户场景

1. **出分第一时间**：刚查到分数，不知道能上什么学校
2. **填报前焦虑**：方案不确定，怕滑档/退档
3. **多方案对比**：有几个选项，不知道哪个更好
4. **政策理解**：对"院校专业组""调剂"等概念不清楚

### 2.3 核心价值主张

| 价值维度 | 传统方式           | 本项目                          | 提升          |
| -------- | ------------------ | ------------------------------- | ------------- |
| 信息获取 | 手工查资料（数天） | 对话即得（分钟）                | 效率提升100x  |
| 政策合规 | 凭经验（易错）     | 自动检查（准）                  | 错误率降低90% |
| 数据精度 | 主观估算           | 2025年位次精确匹配              | 精度提升50%   |
| 个性化   | 标准化话术         | 四维匹配（兴趣+能力+家庭+就业） | 匹配度提升40% |
| 风险提示 | 口头上提醒         | 可视化风险报告                  | 遗漏率降低80% |

### 3.3 功能详情

#### F001：省份自动识别

**用户故事**：

> 作为家长，我希望系统能自动知道我是哪个省的，这样我就不用手动选择省份了。

**验收标准**：

- [x] 可以从对话中自动提取省份名称（如"我是湖南考生"→识别出"湖南"）
- [x] 支持省份简称识别（如"湘"→"湖南"）
- [x] 正确加载对应省份的规则
- [x] 未识别到时给出提示

**技术实现**：

```python
detect_province("我是湖南考生...") # 返回 "湖南"
```

#### F002：方案生成

**用户故事**：

> 作为考生，我希望系统能根据我的分数和性格，推荐适合我的专业和学校。

**验收标准**：

- [x] 支持兴趣测评（霍兰德6型）
- [x] 支持学科强弱评估
- [x] 支持家庭条件匹配
- [x] 输出三套方案（冲/稳/保）
- [x] 每个方案包含6个专业+排序建议

**流程**：

1. 收集信息（5分钟）
2. 分析画像（2分钟）
3. 生成方案（5分钟）
4. 自检输出（1分钟）

#### F003：规范检查

**用户故事**：

> 作为家长，我担心方案有错误导致孩子滑档，希望系统能自动检查。

**验收标准**：

- [x] 检测致命错误（如"45个学校"）
- [x] 检测严重错误（如主观概率）
- [x] 检测一般警告（如体检要求缺失）
- [x] 输出三级分类报告
- [x] 提供具体修正建议

**检查项**：

- 志愿单位是否正确
- 调剂范围是否正确
- 数据是否基于2025年
- 是否标注数据来源

## 5. 用户界面

### 5.1 交互设计

#### 对话式交互（Hermes）

**入口**：

```
/skill gaokao-counselor-long
```

**标准对话流程**：

```
用户：我是湖南考生，578分
龙老师：578分在湖南是位次约26,800名...
      但光知道分数还不够，我得问你几个事：
      1. 你平时最喜欢干啥？...
用户：我喜欢整理数据
龙老师：那就是C型（常规型），适合会计、财务、统计...
      你数学和物理怎么样？...
[对话继续]...
龙老师：好，信息收齐了，我给你生成方案...
      [生成方案中...]
      [自动检查中...]
      你的方案已生成，有问题吗？
```

#### 命令行交互

**规范检查**：

```bash
$ gaokao-checker plan.txt

╔════════════════════════════════════════════════════════╗
║          ✅ 志愿方案规范检查报告                        ║
╠════════════════════════════════════════════════════════╣
║  检测省份：湖南                                         ║
╚════════════════════════════════════════════════════════╝

🔴 【致命错误】
  1. 志愿单位错误
     ❌ 问题：...
     ✅ 修正：...
```

### 5.2 报告设计

#### 可视化报告结构

```
1. 封面
   - 考生信息
   - 咨询时间
   - 规划师（龙老师）

2. 考生画像
   - 基本信息表格
   - 兴趣-能力-家庭-就业四维度雷达图

3. 志愿方案
   - 方案A（冲）：院校+专业+位次+概率
   - 方案B（稳）：...
   - 方案C（保）：...
   - 院校对比决策表

4. 风险提示
   - 致命风险（🔴）
   - 中等风险（🟡）
   - 一般提示（🟢）

5. 行动清单
   - 出分前
   - 出分后
   - 填报时

6. 数据来源
   - 官方网址
   - 数据时效性声明
```
