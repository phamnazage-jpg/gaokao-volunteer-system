## 🎯 系统定位

**不是替代省考试院系统**，而是**前置决策支持服务**：

- 信息整合（招生政策+就业数据+专业解读）
- 智能匹配（考生画像×院校专业库）
- 方案预演（多版本迭代+风险检查）
- 最终填报：引导用户登录省考试院系统实际操作

## 📊 9大场景详细设计

### 场景一：考前认知（高一~高三上学期）

**目标用户**：未参加高考的学生+家长

**核心需求**：

- 了解"我适合什么专业"（兴趣×能力×就业前景）
- 建立目标感（锚定院校层次）
- 学业规划（选科组合与专业覆盖率）

**产品形态**：

```
交互流程：
├── 霍兰德测试（6型人格）
├── 学科强弱自评
├── 家族资源调研
├── 输出：专业方向雷达图
└── 推荐：3个探索方向 + 选科建议

内容资产：
├── 专业百科库（100+专业，每专业100字解读）
├── 就业方向简卡（行业+岗位+薪资区间）
├── 院校层次金字塔
└── 选科覆盖率计算器
```

**技术要点**：

- 无需真实分数，基于预估范围
- 强内容驱动，弱匹配算法
- 引导长期跟踪（关注公众号/加咨询群）

---

### 场景二：考后估分（高考结束~出分前）

**目标用户**：已参加高考，估分阶段

**核心需求**：

- 根据估分做第一轮方案（心理预期管理）
- 提前锁定关注院校（缩小6月25日后的决策范围）
- 熟悉填报规则（避免出分后手忙脚乱）

**产品形态**：

```
交互流程：
├── 估分输入（分科+总分+乐观/悲观区间）
├── 省份+选科确认
├── 目标筛选（地域/专业大类/公办民办）
├── 输出：估分版可行集（A/B/C三档院校名单）
└── 待确认：出分后再细化

关键提示：
⚠️ "估分方案仅供参考，以6月25日真实位次为准"
⚠️ "建议提前登录【阳光高考网】熟悉系统操作"
```

**技术要点**：

- 模糊匹配（估分±20分区间）
- 强调不确定性，避免过度承诺
- 引导用户加"估分交流群"（转化到出分后的服务）

---

### 场景三：深度规划（高意向用户付费/高优先级）

**目标用户**：愿意提供详细信息的考生

**核心需求**：

- 多维度匹配（兴趣+能力+资源+就业+城市偏好）
- 个性化方案（不是通用模板）
- 风险全披露（退档/滑档/调剂/就业红线）

**产品形态**：

```
信息收集中枢（P0/P1/P2/P3四层）：
├── P0 必填（体检+家庭禁区+预算）
├── P1 强烈建议（真实爱好+学习风格+学科强弱）
├── P2 可选（家族资源+地域细化+已有技能）
└── P3 决策层（院校偏好+专业排斥）

输出物：
├── 考生画像报告（PDF，5-8页）
│   ├── 兴趣-能力-资源三维图
│   ├── 学科优势分析
│   ├── 家族资源可利用度
│   └── 风险红线清单
├── v1 基础方案（冲稳保三档）
├── 信息补充卡（如需要，进入v2）
└── v2 精细化方案（迭代后）
```

**技术要点**：

- 信息分层收集（避免一次性问卷过长）
- 多轮对话式收集（类似龙老师风格）
- 版本化管理（v1→v2→...→vn）

### 场景四：出分填报（核心场景）

**目标用户**：已有真实分数+位次

**核心需求**：

- 基于真实位次的精准匹配
- 专业解读（每个推荐专业附100字解读）
- 就业方向建议（3-5个具体岗位）

**产品形态**：

```
输入：
├── 省份/分数/位次（从考试院官方获取）
├── 选科组合
└── 已有补充信息（如之前做过深度规划）

输出方案格式：

【考生基础信息卡】
├─ 省份/分数/位次/选科
├─ 霍兰德类型/学科强弱
└─ 家庭约束条件

【冲稳保三方案】（每档3-5个院校专业组）
├─ 院校专业组名称+2025参考位次
├─ 6个专业排序（附100字专业解读）
├─ 组内服从建议（是/否+理由）
├─ 就业方向建议（3-5个岗位+起薪区间）
└─ 风险提示（体检/单科/调剂/退档）

【专业解读示例】
专业：网络与新媒体
解读：融合新闻学、传播学与数字技术，培养新媒体运营、内容策划、短视频制作人才。课程包含传播学概论、新媒体运营、数据分析、视频剪辑等。适合有创作热情、熟悉互联网生态的学生。
就业方向：新媒体运营（8-15K）、内容策划（10-20K）、短视频编导（12-25K）、自媒体创业（收入波动大，头部月入5万+）、品牌公关（8-18K）。
```

**技术要点**：

- 专业解读库（100+专业，每专业100字）
- 就业数据库（岗位+薪资+地域+趋势）
- 输出格式标准化（便于家长阅读）

### 场景五：考生方案版本积累

**目标**：同一考生的历史方案可追溯、可对比

**数据模型**：

```yaml
# 考生档案（以身份证/手机号唯一标识）
candidate_id: "440106200809123456" # 匿名化存储

versions:
  - version: "v1_20260315" # 时间戳+版本
    stage: "考前认知" # 场景标记
    data_snapshot:
      grade: "高三上学期"
      estimated_score: 550
      interests: ["写作", "视频剪辑"]
    output: "专业方向雷达图.png"

  - version: "v2_20260609" # 考后估分
    stage: "考后估分"
    data_snapshot:
      estimated_score: 560
      score_range: [540, 580]
    output: "估分版可行集.pdf"

  - version: "v3_20260625" # 真实出分
    stage: "深度规划"
    data_snapshot:
      real_score: 550
      real_rank: 16000
      p0_info: { 体检: "正常", 家庭禁区: ["警校"] }
      p1_info: { 爱好: "公众号", 资源: "父亲教育局" }
    output: "v2精细化方案.pdf"

  - version: "v4_20260628" # 最终填报版
    stage: "出分填报"
    final_decision: # 用户最终选择
      school: "广州大学"
      major: "网络与新媒体"
    outcome: # 实际录取结果（后续回填）
      admitted: true
      school: "广州大学"
      major: "新闻学" # 可能调剂

feedback: #  outcome回填
  satisfaction: 4 # 1-5星
  comment: "被调剂到新闻学，但还算满意"
```

**技术要点**：

- 版本号规范：`vN_YYYYMMDD`
- 数据快照（记录当时的输入，避免覆盖）
- 录取结果回填（用于系统优化）

### 场景七：统计信息中心

**目标**：为运营者提供数据洞察，为算法优化提供依据

**统计维度**：

```
1. 用户画像统计
├── 按省份分布（广东/湖南/...）
├── 按分数段分布（<500/500-550/550-600/>600）
├── 按选科组合分布
└── 按咨询阶段分布（考前/估分/出分）

2. 服务效率统计
├── 平均响应时间（首响/方案生成/v2迭代）
├── 方案通过率（v1→v2转化率）
├── 用户满意度（1-5星）
└── 复购/推荐率

3. 专业热度统计
├── 被咨询最多的专业Top20
├── 录取率最高的专业Top20
├── "后悔率"最高的专业（毕业后反馈）
└── 新兴专业趋势（AI相关/新能源/养老）

4. 系统效能统计
├── 知识库命中率（本地数据满足率）
├── 平均对话轮数
├── 最常见问题Top10
└── 边界案例数（需人工介入的比例）
```

**输出物**：

- 日报（关键指标看板）
- 周报（趋势分析+异常预警）
- 季报（深度洞察+产品优化建议）

### 场景八：双格式交付（PDF + 单页网页）

**目标**：满足不同场景的阅读需求

**PDF 版**：

```
用途：打印/存档/家长传阅
格式：A4竖版，5-10页
特点：
├── 正式感强（类似正式报告）
├── 支持分页目录
├── 可手写批注
└── 适合老年家长阅读

技术实现：
├── Markdown → Pandoc → PDF
├── wkhtmltopdf引擎
├── 中文排版优化
└── 自动生成页眉页脚
```

**单页网页版**：

```
用途：手机阅读/微信分享/快速预览
格式：单HTML文件，响应式
特点：
├── 手机端友好（字号自适应）
├── 可交互（展开/收起详细信息）
├── 一键分享（生成短链接/二维码）
├── 视觉化（雷达图+对比表）
└── 可嵌入链接（跳转到阳光高考网）

技术实现：
├── HTML5 + Tailwind CSS
├── Chart.js可视化
├── 内联所有资源（单文件）
└── 支持打印为PDF（备用）
```

**默认交付策略**：

```
普通用户 → 单页网页（主推，体验好）
付费用户/深度规划 → PDF + 单页网页（双格式）
老年家长 → PDF（打印版）
```

### 场景九：移动端咨询与方案查看 ⭐新增

**目标**：手机端全流程支持（咨询→信息收集→方案查看）

**背景**：

- 70%以上家长/学生用手机咨询（微信/WhatsApp/短信）
- PC端咨询场景受限（工作/工作时间用电脑不方便）
- 需要支持沉浸式长对话（30-50轮）
- 方案查看以大屏为主，但手机端快速预览需求高

**移动端交互设计原则**：

```
┌─────────────────────────────────────────────┐
│  移动端优化原则                              │
├─────────────────────────────────────────────┤
│ 1. 碎片化友好：支持随时中断、随时恢复        │
│ 2. 输入法友好：减少长文本输入，多用选择/按钮 │
│ 3. 信息密度控制：单屏不超过3个信息点        │
│ 4. 异步处理：复杂计算后台跑，不阻塞对话     │
│ 5. 多模态：支持语音输入（长辈友好）         │
│ 6. 方案展示：卡片式+折叠式（避免长文本）    │
└─────────────────────────────────────────────┘
```

**移动端专属功能**：

```
📱 咨询入口（WeChat/WhatsApp/短信）
├── 快捷指令
│   ├── "/开始" → 快速开始咨询
│   ├── "/继续" → 恢复上次未完成方案
│   ├── "/我的方案" → 查看历史方案
│   └── "/帮助" → 使用指南
│
├── 交互优化
│   ├── 分段收集：每次只问1-2个问题
│   ├── 预设选项：数字回复（1/A/B/C）代替打字
│   ├── 进度提示："第3/7步·还剩2分钟"
│   ├── 保存点：每5句自动保存，崩溃可恢复
│   └── 超时提醒：30分钟无回复发提醒
│
└── 语音支持（长辈模式）
    ├── 语音提问转文字
    ├── 文字回复转语音（播报）
    └── 方言识别（粤语/湖南话等）

📊 方案查看（H5页面）
├── 瀑布流卡片（每校一张卡片）
├── 左右滑对比（类似Tinder滑动对比）
├── 展开/收起详情（避免信息过载）
├── 一键分享（生成长图/二维码）
├── 收藏夹（长按收藏院校）
└── 离线查看（保存到本地缓存）
```

**移动端信息收集流程（示例）**：

```
【第1轮】龙老师："你好！我是高考志愿规划师龙老师。先问你：孩子哪个省参加高考？直接回复省份名就行~"

【用户】"广东"

【第2轮】龙老师："广东！了解了。下一个问题：孩子擅长哪两门学科？回复数字就行~
1️⃣ 语文/政治/历史
2️⃣ 数学/物理
3️⃣ 英语/外语
4️⃣ 化学/生物
5️⃣ 地理/其他"

【用户】"1"

【第3轮】龙老师："政治历史很强！👍 再选一门强的~
..."
```

**移动端技术方案**：

```
┌─────────────────────────────────────────────┐
│           移动端技术栈                        │
├─────────────────────────────────────────────┤
│ 渠道接入层                                   │
```

**移动端与PC端差异**：

| 维度       | PC端                      | 移动端                 |
| ---------- | ------------------------- | ---------------------- |
| 对话轮数   | 可一次性长对话（20-30轮） | 碎片化（每次3-5轮）    |
| 输入方式   | 打字为主                  | 选择为主，语音为辅     |
| 方案展示   | 多窗格对比（Excel式）     | 卡片滑动对比           |
| 阅读深度   | 可精读10页PDF             | 快速浏览+收藏          |
| 分享方式   | 邮件/网盘                 | 微信一键分享           |
| 长辈友好度 | 低                        | 高（字体大+语音+简单） |

**多端数据同步**：

```yaml
数据同步协议：
├── 核心数据：云端（加密）
│   ├── 考生档案
│   ├── 方案版本
│   └── 咨询历史
├── 缓存策略：
│   ├── 移动端：本地SQLite（离线可查看）
│   ├── PC端：本地JSON文件
│   └── 同步触发：每次对话结束自动同步
└── 冲突解决：
├── 时间戳优先（最新覆盖）
└── 版本链（可回滚到任意版本）
```
