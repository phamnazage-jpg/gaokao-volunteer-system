# 管理后台设计

## 管理端系统架构与功能

**版本**: v1.0  
**状态**: 已落地（MVP 完成，持续演进）  
**最后更新**: 2026-06-13

---

## 1. 概述

### 1.1 为什么要管理端？

| 需求     | 解决什么问题               |
| -------- | -------------------------- |
| 用户管理 | 查看注册用户、使用情况     |
| 内容管理 | 管理案例、规则、模板       |
| 数据监控 | 了解系统运行状态、用户行为 |
| 运营支持 | 协助用户、处理问题         |
| 质量把控 | 监控方案质量、错误检测     |

### 1.2 使用角色

| 角色           | 权限级别 | 主要操作                     |
| -------------- | :------: | ---------------------------- |
| **超级管理员** | 所有权限 | 创建管理员、系统配置         |
| **产品经理**   |    高    | 查看数据、导出报告、配置功能 |
| **运营人员**   |    中    | 处理用户反馈、管理内容       |
| **数据分析师** |    中    | 查看统计、导出数据           |
| **客服**       |    低    | 查看用户信息、协助问题       |

## 2. 管理端架构

### 2.1 技术方案

```
管理端架构:
├── 前端
│   ├── React/Vue (单页应用)
│   ├── Ant Design / Element Plus (UI组件库)
│   └── 响应式（支持PC端）
│
├── 后端 API
│   ├── RESTful API
│   ├── 鉴权：JWT Token
│   └── RBAC 权限控制
│
└── 数据库
    ├── 用户数据（只读）
    ├── 统计数据（聚合）
    └── 操作日志
```

### 2.2 安全设计

- ✅ **独立域名/路径**: admin.gk.example.com / /admin
- ✅ **IP白名单**: 限制访问IP（可选）
- ✅ **双因素认证**: 管理员登录需2FA
- ✅ **操作审计**: 所有操作记录日志
- ✅ **数据脱敏**: 用户敏感信息脱敏显示

#### 核心指标

| 指标            | 计算方式        | 告警阈值         |
| --------------- | --------------- | ---------------- |
| 日活跃用户(DAU) | 当日访问用户数  | <50触发关注      |
| 新用户注册      | 当日注册用户    | <10触发关注      |
| 方案生成数      | 当日生成方案数  | 趋势下降>20%告警 |
| 规范检查错误率  | 错误数/总检查数 | >10%触发告警     |
| 平均响应时间    | API响应时长     | >3s触发告警      |
| 系统错误率      | 错误请求/总请求 | >5%触发告警      |

## 5. 技术实现

### 5.1 前后端分离

```
管理端技术栈:
├── 前端
│   ├── React 18 / Vue 3
│   ├── Ant Design Pro / Element Plus
│   ├── ECharts (数据可视化)
│   └── Axios (HTTP请求)
│
└── 后端
    ├── Python FastAPI / Node.js Express
    ├── JWT + RBAC
    ├── 接口与Web端复用
    └── 独立管理端鉴权
```

### 5.2 部署方案

#### 方案A：独立部署（推荐）

```
admin.gk.example.com
├── 独立前端构建
├── 独立后端服务
└── 独立域名（安全隔离）
```

#### 方案B：集成部署

```
gk.example.com/admin
├── 与Web端共享后端
├── 通过路由区分权限
└── 配置IP白名单
```

## 7. 决策记录

#### D001: 管理端是否单独部署？

**决策**: 独立域名部署（admin.gk.example.com）
**理由**: 安全隔离，便于权限控制

#### D002: 用什么前端框架？

**决策**: Ant Design Pro (React)
**理由**: 成熟的后台管理模板，中文文档完善

#### D003: 是否支持数据导出？

**决策**: 支持，但需审批
**理由**: 脱敏后的汇总数据可以导出，保护用户隐私
