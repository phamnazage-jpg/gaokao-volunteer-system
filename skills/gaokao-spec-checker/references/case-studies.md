## 案例1：原始方案（错误版）— 致命错误密集

### 错误内容（简化）
```
湖南578分考生志愿方案

本次共填报45个学校志愿：
志愿01：江西财经大学，会计学
志愿02：长沙理工大学，财务管理
...
志愿45：长沙师范学院，财务管理

录取概率：
志愿01 35%
志愿02 45%
...
志愿45 90%

投档后如果不服从调剂，会被退到下个志愿。

所有会计学专业都要求物理+化学+生物选科。
```

### 检查器诊断

| 错误 | 严重性 | 规则 |
|------|--------|------|
| 志愿单位："45个学校" | 🔴 致命 | 应为"45个院校专业组" |
| 投档后规则错误 | 🔴 致命 | 应为"退档进入征集志愿" |
| 主观概率 | 🟡 严重 | 35%、80%无数据基础 |
| 选科一刀切 | 🟡 严重 | 应逐校核实 |
| 体检要求缺失 | 🟢 中等 | 需提示 |
| 专业代码缺失 | 🟢 中等 | 需提供 |

**检查器报告输出**：
- 🔴 致命错误：2个
- 🟡 严重错误：2个
- 🟢 一般警告：2个
- ❌ 评价：必须立即修改

## 复盘教训（防止同类错误复发）

### 错误的根本原因

| 错误类型 | 根因 | 防止方法 |
|---------|------|---------|
| 志愿单位错误 | 政策更新滞后（用2024年旧概念） | 每年政策更新时重新学习 |
| 主观概率 | 没有基于真实位次数据 | 绝不编造，只提供位次参考 |
| 选科一刀切 | 批量假设，不逐校核实 | 强制要求"逐校核实" |
| 退档规则错误 | 平行志愿三大原则记混 | 背诵"分数优先、遵循志愿、一次投档" |
| 体检缺失 | 风险意识弱 | 必须前置提示 |

### 工作流教训

**错误流**：生成方案 → 提交给用户 → 用户发现错误 → 返工

**正确流**：
1. 收集信息
2. 明确数据时效（2025年参考，2026年待公布）
3. 按"院校专业组"生成方案
4. 标注每组选科要求
5. ⭐ **调用 gaokao-spec-checker 自检**
6. 修正致命/严重错误
7. 输出方案 + 风险提示 + 行动清单
8. 强调"以官方为准"

## 错误检测规则（自动检查脚本用）

### 致命错误规则（regex）
```python
# 志愿单位
r"45个学校" / r"45所学校" / r"45个院校" → 🔴 致命

# 投档后规则
r"(退档后|被退).*?(下个志愿|下一个|其他志愿)" → 🔴 致命

# 征集志愿次数
r"本科.*1次.*征集" → 🔴 致命
```

### 严重错误规则
```python
# 主观概率
r"\d{2,3}\s*%.*?(录取|概率|机会|把握)" → 🟡 严重

# 选科一刀切
r"会计.*?物.*?化.*?生" → 🟡 严重

# 位次数据年份
r"2024年位次" / r"2023年位次" → 🟡 严重（应使用2025年）
```

### 一般警告规则
```python
# 缺少代码
not in text "代码" and in text "院校" → 🟢 中等
not in text "代码" and in text "专业" → 🟢 中等

# 缺少体检
not in text "体检" and in text "风险" → 🟢 中等
```
