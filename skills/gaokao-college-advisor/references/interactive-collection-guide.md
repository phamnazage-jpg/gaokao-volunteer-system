# 交互式信息收集系统 - 使用指南

## 系统概述

交互式信息收集系统采用**分步骤引导式交互**，确保收集完整、准确的考生信息，为志愿填报提供数据基础。

**核心模块**：
1. **完整交互脚本** (`gaokao-collect-info.py`): 7步问答式收集
2. **快速对话模板** (`gaokao-quick-collect.py`): 可直接复制使用的问卷

---

## 7步收集流程

| 步骤 | 内容 | 核心问题 |
|------|------|----------|
| **1** | 基本信息 | 姓名、省份 |
| **2** | 高考信息 | 模式、分数、位次、选科 |
| **3** | 兴趣测评 | 霍兰德6型测试 |
| **4** | 能力评估 | 学科强弱、软技能 |
| **5** | 职业目标 | 优先级、毕业规划 |
| **6** | 家庭背景 | 经济、地域、资源 |
| **7** | 偏好设置 | 院校/专业优先级、调剂意愿 |

---

## 霍兰德兴趣测试 (RIASEC)

| 类型 | 特征 | 适合专业 | 问题对应 |
|------|------|---------|---------- |
| **R现实型** | 动手操作、技术 | 计算机、机械、电子 | Q1-A, Q2-A |
| **I研究型** | 分析探索 | 数学、物理、医学 | Q1-B, Q2-B |
| **A艺术型** | 创作设计 | 设计、传媒、建筑 | Q1-C, Q2-C |
| **S社会型** | 助人服务 | 师范、医学、社工 | Q1-D, Q2-D |
| **E企业型** | 领导管理 | 商科、管理、法律 | Q1-E, Q2-E |
| **C常规型** | 规范有序 | 财务、行政、统计 | Q1-F, Q2-F |

## 快速使用

### 方式1：运行完整脚本

```bash
python3 ~/.local/bin/gaokao-collect-info.py
```

### 方式2：复制对话模板

直接发送给用户：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【步骤1/7】基本信息
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. 考生姓名：__________
2. 所在省份：__________
3. 高考模式：①传统文理 ②3+3 ③3+1+2

【步骤2/7】高考信息
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
4. 高考总分：__________分
5. 全省位次：__________名
6. 选科组合：__________

【步骤3/7】兴趣测评
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
7. 课余最喜欢做什么？
   A.拆解东西 B.研究问题 C.创作设计 
   D.社交活动 E.策划组织 F.整理规划

8. 如果完全不用担心钱，最想做什么？（同上）

9. 明确不喜欢什么？（多选）
   A.背诵 B.数学 C.社交 D.实验 E.代码

💡 填写后发回即可
```

```json
  "career_goals": {"plan": "本科就业"},
  "family_background": {"economic": "中等"},
  "preferences": {"school_priority": "专业优先"}

```

## 质量检查

- [ ] 信息完整度：7个模块都已填写
- [ ] 数据准确性：位次、分数已核对
- [ ] 逻辑一致性：能力评估与兴趣类型匹配
- [ ] 风险识别：有明确的不喜欢项记录
